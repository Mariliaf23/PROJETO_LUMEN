# Plano de Melhorias - Projeto LUMEN (Biblioteca Escolar)

## Diagnóstico Atual

O app já tem: login, dashboard com gráficos, CRUD de livros, exemplares, empréstimos, reservas, multas, gerenciamento de usuários e configurações. Porém há bugs críticos, funcionalidades incompletas e código morto.

---

## FASE 1 — Corrigir Bugs Críticos

### 1.1 Funções duplicadas em `database_config.py`
- `cadastrar_usuario` definida 2 vezes (linhas ~33 e ~685)
- `listar_usuarios` definida 2 vezes (linhas ~53 e ~705)
- **Ação**: Remover as duplicatas antigas, manter apenas as versões atualizadas

### 1.2 Senhas em texto plano
- Senhas salvas diretamente no banco sem hash
- **Ação**: Usar `hashlib.sha256` para hashear senhas ao cadastrar e verificar no login

### 1.3 Emprestimos desempacota 7 colunas mas query retorna 8
- `DetalheEmprestimo` e `_criar_item_emp` desempacotam数量 incorreta
- **Ação**: Corrigir desempacotamento para 8 colunas

### 1.4 Login busca por `nome` mas campo diz "Usuario"
- **Ação**: Alterar para buscar por `nome` ou `email` (aceitar ambos)

---

## FASE 2 — Funcionalidades Ausentes

### 2.1 Tela de Edição de Livros
- Atualmente só é possível cadastrar e excluir livros
- **Ação**: Adicionar botão "Editar" na lista de livros → abre formulário com dados preenchidos

### 2.2 Integração de Autores nos Livros
- Tabela `livro_autor` existe no schema mas não é usada na UI
- **Ação**: Na tela de livros, adicionar combo para selecionar/autores associados

### 2.3 Botão de Logout
- Não existe forma de sair da conta
- **Ação**: Adicionar botão "Sair" na sidebar do dashboard (no topo ou rodapé)

### 2.4 Reserva → Empréstimo automático
- Reservas ficam ativas mas nunca convertem em empréstimo
- **Ação**: Ao finalizar um empréstimo, verificar se há reserva ativa para o mesmo livro e notificar o bibliotecário

## FASE 3 — Módulos Vazios

### 3.1 `validador.py` — Validações de entrada
- **Ação**: Implementar validações:
  - Email/celular: formato básico
  - ISBN: 13 dígitos
  - Campos obrigatórios: não vazios
  - Data: formato AAAA-MM-DD válido

### 3.2 `report_gen.py` — Geração de Relatórios PDF
- `fpdf2` já está no requirements.txt
- **Ação**: Criar função para gerar PDF com:
  - Lista de livros cadastrados
  - Empréstimos ativos
  - Usuários com multas pendentes
- Adicionar botão "Gerar Relatório" na tela de configurações ou dashboard

## FASE 4 — Melhorias de UI/UX

### 4.1 Notificação padronizada
- Cada tela usa posicionamento diferente para notificações
- **Ação**: Criar helper `mostrar_notificacao(parent, mensagem)` centralizado

### 4.2 Botão "Voltar" padronizado
- **Ação**: Usar sempre `criar_botao()` (outline) para voltar, em todas as telas

### 4.3 Limpar código morto
- `tela_devolucoes.py` — lógica duplicada em `emprestimos.py` (já integrada)
- `transitions.py` — stub obsoleto
- `tela_cadastro_login.py` — cadastro público não integrado ao fluxo
- `teste.py` — protótipo avulso
- **Ação**: Remover ou marcar como deprecated

### 4.4 Busca e filtro em mais telas
- Empréstimos não tem busca por nome/título
- Exemplares não tem busca
- **Ação**: Adicionar campo de busca nas telas de empréstimos e exemplares

## FASE 5 — Segurança e Dados

### 5.1 Dados de exemplo inconsistentes
- `schema.sql` insere 21 categorias com acentos
- `conector.py` insere 5 categorias sem acentos
- **Ação**: Manter apenas o `schema.sql` para dados iniciais, remover `_inserir_dados_exemplo`

### 5.2 Status 'bloqueado' nunca utilizado
- **Ação**: Implementar bloqueio automático após X multas pendentes, ou remover do ENUM
