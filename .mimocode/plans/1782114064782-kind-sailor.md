# Plano: Controle de Acesso por Tipo de Usuário

## Contexto
O sistema atualmente permite que todos os usuários logados acessem todas as telas. O objetivo é implementar controle de acesso baseado em dois tipos de usuário: **Diretor** (acesso total) e **Bibliotecário** (acesso limitado).

## Tipos de Usuário e Permissões

### Diretor
- Acesso a **todas** as telas do sistema
- Pode gerenciar usuários, configurações, e todas as operações

### Bibliotecário
| Tela | Acesso |
|------|--------|
| Dashboard | ✅ Visualização |
| Livros | ✅ Cadastro/Edição |
| Exemplares | ✅ Gerenciamento |
| Empréstimos | ✅ Cadastro + Mudar status |
| Devoluções | ✅ Registro |
| Cadastro de Alunos | ✅ Apenas criar novos |
| Gerenciar Usuários | ❌ Sem acesso |
| Configurações | ❌ Sem acesso |

## Alterações Necessárias

### 1. Banco de Dados (`database/schema.sql`)
- Alterar ENUM `tipo_usuario` de `('aluno', 'professor', 'funcionario', 'bibliotecario')` para `('diretor', 'bibliotecario')`

### 2. Usuário Padrão (`services/conector.py`)
- Alterar o usuário padrão de `bibliotecario` para `diretor`

### 3. Tela de Login (`screen/tela_login.py`)
- Não precisa de alteração (já usa `tipo_usuario`)

### 4. Tela de Cadastro de Login (`screen/tela_cadastro_login.py`)
- Remover auto-cadastro como `bibliotecario` (segurança)
- ou bloquear acesso não-autenticado

### 5. Dashboard (`screen/dashboard.py`)
- **Sidebar**: Filtrar botões baseado no `tipo_usuario`
  - Bibliotecário: DASHBOARD, LIVROS, EXEMPLARES, EMPRESTIMOS, DEVOLUCOES, CADASTRO USUÁRIO (só alunos)
  - Diretor: Todos os botões

### 6. TelaCadastroUsuario (`screen/tela_cadastro_usuario.py`)
- Bibliotecário: Só pode cadastrar tipo `aluno`
- Diretor: Pode cadastrar qualquer tipo

### 7. Validação de Acesso
- Criar função `verificar_acesso(tela_necessaria)` em `services/` ou `services/app_controller.py`
- Chamar antes de navegar para telas restritas
- Redirecionar para dashboard se sem acesso

## Arquivos para Modificar
1. `database/schema.sql` - ENUM de tipos
2. `services/conector.py` - Usuário padrão
3. `services/app_controller.py` - Função de verificação de acesso
4. `screen/dashboard.py` - Sidebar condicional
5. `screen/tela_cadastro_usuario.py` - Restrição de tipo
6. `screen/tela_cadastro_login.py` - Segurança de auto-cadastro

## Verificação
1. Criar usuário com tipo `diretor` → deve ver todas as telas
2. Criar usuário com tipo `bibliotecário` → deve ver apenas telas permitidas
3. Tentar acessar tela restrita por URL direta → deve ser redirecionado
4. Verificar sidebar mostra apenas opções permitidas
5. Verificar cadastro de usuário limitado a `aluno` para bibliotecário
